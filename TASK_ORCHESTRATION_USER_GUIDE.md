# Руководство пользователя: Система оркестрации задач

## Обзор

Система оркестрации задач позволяет сотрудникам создавать задачи с прикреплением документов и организовывать их в канбан-досках с обязательным назначением исполнителей и возможностью одобрения.

## Основные возможности

### 📋 Создание задач
- **Обязательное назначение**: Каждая задача должна быть назначена исполнителю
- **Прикрепление документов**: Возможность прикрепить документы к задаче
- **Типы задач**: Общие задачи или задачи, требующие одобрения

### 🔄 Рабочий процесс одобрения
- **Инициация**: Сотрудник создает задачу и выбирает одобряющего
- **Проверка**: Одобряющий получает уведомление и может одобрить или отклонить
- **Уведомления**: Все участники получают уведомления о статусе задачи

### 📊 Канбан-доски
- **Группировка**: Задачи можно группировать по исполнителю, отделу или логической группе
- **Статусы**: pending → in_review → approved/declined
- **Перемещение**: Перетаскивание задач между колонками

## Как использовать

### Создание новой задачи

1. **Перейдите на страницу задач** в разделе "Дашборд" → "Задачи"
2. **Выберите организацию и доску** из выпадающих списков в фильтрах
3. **Нажмите "Создать задачу"** в заголовке доски
4. **Заполните форму**:
   - **Название задачи** (обязательно, минимум 3 символа)
   - **Описание** (необязательно)
   - **Исполнитель** (обязательно выбрать из списка участников доски)
   - **Тип задачи**:
     - "Общая задача" - обычная задача без одобрения
     - "Требует одобрения" - требует выбора одобряющего
   - **Одобряющий** (обязательно для задач с одобрением)
   - **Срок выполнения** (необязательно)
   - **Прикрепленные документы** (необязательно, выбрать из доступных)

5. **Нажмите "Создать"** для сохранения задачи

### Работа с канбан-доской

#### Группировка задач
- **По исполнителю**: Задачи группируются по назначенным пользователям
- **По подразделению**: Группировка по отделам исполнителей
- **По группе задач**: Логическая группировка задач

#### Перемещение задач
- **Назад/Вперед**: Используйте кнопки для перемещения между статусами
- **Ограничения**: Некоторые переходы могут быть заблокированы бизнес-правилами

### Одобрение задач

#### Для одобряющих
1. **Найдите задачу** в колонке "На проверке" (in_review)
2. **Изучите задачу**: Название, описание, прикрепленные документы
3. **Примите решение**:
   - **Одобрить**: Задача перейдет в статус "Одобрено"
   - **Отклонить**: Задача перейдет в статус "Отклонено", можно добавить комментарий

#### Для инициаторов
- **Следите за статусом** задачи в канбан-доске
- **Получайте уведомления** об изменениях статуса
- **Просматривайте комментарии** одобряющих при отклонении

### Прикрепление документов

#### При создании задачи
- В форме создания задачи выберите "Прикрепленные документы"
- Отметьте нужные документы из списка доступных
- Документы должны быть в статусе "Опубликован" для прикрепления

#### Просмотр прикреплений
- В деталях задачи отображаются все прикрепленные документы
- Можно перейти к просмотру или скачиванию документов

## Статусы задач

| Статус | Описание | Действия |
|--------|----------|----------|
| **Ожидает проверки** | Задача создана, ожидает обработки | Назначение исполнителя, перемещение |
| **На проверке** | Задача отправлена на одобрение | Одобрение/отклонение (для одобряющих) |
| **Одобрено** | Задача одобрена | - |
| **Отклонено** | Задача отклонена | - |

## Уведомления

Система отправляет уведомления участникам процесса:

- **Инициатору задачи**: О создании, изменении статуса, решениях одобряющих
- **Одобряющим**: О поступлении задач на одобрение
- **Исполнителям**: О назначении задач

## Советы по использованию

### Для эффективной работы
- **Четкие названия**: Используйте понятные названия задач
- **Подробные описания**: Добавляйте контекст и требования
- **Правильный выбор одобряющих**: Выбирайте компетентных сотрудников
- **Сроки**: Устанавливайте реалистичные сроки выполнения

### Избегайте распространенных ошибок
- Не создавайте задачи без назначения исполнителя
- Проверяйте прикрепленные документы перед отправкой на одобрение
- Регулярно проверяйте статус задач, требующих вашего одобрения

## Поддержка

При возникновении вопросов или проблем:
1. Проверьте это руководство
2. Обратитесь к администратору системы
3. Создайте задачу в системе поддержки
